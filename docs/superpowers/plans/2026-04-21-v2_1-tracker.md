`docs/superpowers/plans/2026-04-21-v2_1-implementation.md` with `- [ ]`
checkboxes per TDD step. This file carries **PR-level** state (branch,
HEAD, GitHub PR number, review status) so a fresh session after `/new`
can resume in one hop without re-scanning every checkbox.

corresponding row here. Every merge clears the `head` column and sets
`status = merged <sha-on-main>`. Tracker lives on `main` so it is
always the current truth regardless of feature-branch activity.
